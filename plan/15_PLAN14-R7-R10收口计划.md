# PLAN-14 R7–R10：收口计划

## 0. 背景与缺口诊断

PLAN-14 M0–M5 已完成并推送到 master，但诚实核查发现 3 项 DoD 过度标记、M6/M7 整块未做。

**过度标记（已打勾但只做了一半）：**
- "新 Turn/Lease/预算均能抢占"——`should_preempt_step()` 纯函数 + signal 表存在，但 `drift_runner.py` 是**单次执行**，从未在运行中调用抢占检查
- "paused/waiting 可从兼容 Checkpoint 恢复"——只有 `validate_checkpoint_for_resume` 校验器，无实际 resume 流程

**整块未做：** M6（结果投影+Dashboard+反馈）、M7（回放/指标/门禁/安全测试）

## 2. R7：M5 执行接线

### 目标

让 `drift_runner.py` 具备"多步执行 + 每步前抢占检查 + 安全点写 Checkpoint + 真正的 resume"，使 3 项过度标记的 DoD 做实。

### 任务

1. **重写 `handle_drift_run` 为多步循环：**
   - 步数限制 `manifest.max_steps`、时间限制 `manifest.max_runtime_seconds`
   - 每步前调用 `should_preempt_step()`（lease_valid / cancel / turns / backlog / budget）
   - 被抢占 → 在安全点写 `DriftCheckpointV1`（含 cursor/budget_used/completed_actions），更新 `drift_runs.status=paused` + `preemption_reason`，**释放 Lease**（通过 dispatcher 放弃当前 attempt）
   - 步数/时间耗尽 → 进入 wrap-up（仅允许 write-checkpoint / finish）

2. **每步 budget 累计（跨 Attempt 不重置）：**
   - Task/Attempt 已有 `retry_policy`；budget 消耗记入 `drift_runs` 投影行 `result_ref` 或扩展列
   - 恢复后读取已用 budget，新 attempt 累加

3. **真正的 resume 流程：**
   - 新增 `resume_drift_run(conn, run_id, ctx)`：读 checkpoint JSON → `validate_checkpoint_for_resume()` → 不兼容则置 `needs_review`；兼容则从 `step_index+1` 续跑
   - 接入 TaskHandler：drift.run 若检测到 `drift_runs.status=paused` 且有 result_ref checkpoint，则走 resume 路径
   - 校验 config_version_id / skill_version / checkpoint_schema_version

4. **Lease 接线：**
   - drift.run 作为普通 Task 已被 TaskWorker 的 heartbeat_loop 保活；抢占时应让 Lease 在安全点过期或被显式释放
   - 每步续约失败 → 立即停止领取新 step

5. **Skill 注册：** `handle_drift_run` 在 resume 时重用原 skill_name/skill_version（不重新选择）

### Schema

复用现有 `drift_runs.result_ref`（存 checkpoint JSON 引用）+ `drift_runs.preemption_reason`。checkpoint 实际 JSON 存为 payload 风格字符串——新增列：

```sql
ALTER TABLE drift_runs ADD COLUMN budget_used_json TEXT NOT NULL DEFAULT '{}';
ALTER TABLE drift_runs ADD COLUMN steps_taken INTEGER NOT NULL DEFAULT 0;
```

### 主要文件

- `src/cogito/service/drift_runner.py`（重写核心循环）
- `src/cogito/service/drift_preemption.py`（扩展 write_checkpoint 调通）
- `src/cogito/store/drift_repo.py`（扩展 update_status 支持 budget/steps）
- `src/cogito/store/migrations/0050_drift_run_progress.sql`（budget/steps 列）
- `tests/drift/test_runner_loop.py`（新测试）

### 必测场景（M5 验收）

- step 执行前到达新 Turn → 下一 step 前暂停，status=paused，preemption_reason=preempted_by_turn
- step 执行中 budget 耗尽 → wrap-up，进 paused:budget_exhausted
- Checkpoint 写入后进程退出（模拟崩溃）→ 重启后 resume 从 step_index+1 续跑，不重放 completed_actions
- Skill 升级导致 checkpoint schema 不兼容 → needs_review，不静默继续
- 恢复后 budget 累计不重置（step N 的 budget = 前 N-1 步之和 + 本次）
- Lease 续租失败 → 零新副作用
- config_version 变化 → resume 校验拒绝

### 验收

- DR-P0-03 完全关闭（含端到端测试证据）
- 新 Turn 到达后在一个安全 step 内停止 Drift
- 崩溃恢复不重复已确认副作用
- paused 可从兼容 Checkpoint 恢复（真正的 resume 流程存在并测试通过）

4～6 个开发日。

## 5. R10：M7 — 回放、指标、门禁、安全测试

### 目标

- Drift：有工作/无工作/连续 paused/失败退避
- 新 Turn 抢占、重启/misfire/Lease 丢失
- Prompt injection 内容、模型非法 JSON、工具越权

### 任务 B：核心指标

- Drift：admission_rate / no_value_rate / completion_rate / paused_recovery_rate / preemption_latency_p95 / duplicate_side_effect_count（断言=0）/ unauthorized_tool_execution_count（断言=0）/ model_cost_per_useful_result
