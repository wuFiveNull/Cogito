# 模型适配层

## 1. 边界

Model Adapter 将统一契约转换为 Provider SDK，不承担 Prompt 策略、Tool 授权、记忆写入或 Channel 流式投递。

## 2. ModelRequest

```text
request_id
model_role
messages[{role,content_parts,trust_label}]
tools[{name,description,input_schema}]
response_schema
temperature/top_p
max_output_tokens
stop
stream
timeout
provider_options
trace_context
```

Adapter 必须拒绝无法安全转换的 ContentPart 或 Tool Schema，不能静默丢字段。

## 3. ModelResponse

```text
request_id
provider_request_id
model_id
content_parts
tool_calls
structured_output
finish_reason
usage{input,output,cached,reasoning?}
latency
raw_response_ref
```

统一 `finish_reason`：`stop | tool_calls | length | content_filter | cancelled | error`。

## 4. StreamEvent

```text
response_started
content_delta
tool_call_started
tool_argument_delta
tool_call_completed
usage_updated
response_completed
response_error
```

事件具有单调 `sequence_no`。Adapter 负责把 Provider 增量组装为最终 ModelResponse；Agent Loop 不依赖 SDK 私有事件。

## 5. 能力声明

```text
context_window
max_output_tokens
modalities
supports_streaming
supports_tools
supports_parallel_tools
supports_json_schema
supports_prompt_cache
tool_schema_limits
```

能力在启动和定期健康检查时缓存，调用失败可以刷新，但不能由单次模型文本改变。

视觉调用使用独立 `model_role=vlm`。Provider 只有在 `modalities` 明确包含
`image` 时才能接收 `image_url` 内容块；角色缺失时不得把图片静默发给 text-only
`main`。Data URI 由 Core-owned Payload 在调用时生成，不在 Context 和普通 Trace
中持久化。

## 6. Tool Schema 转换

- 名称映射必须可逆并保存映射表；
- 不支持的 JSON Schema 关键字在调用前报错；
- Provider 返回的参数统一解析后再次用原 Schema 校验；
- Tool description 受长度限制并保留安全约束；
- 多个 Tool 名称冲突在 Registry 阶段解决。

## 7. 错误映射

```text
authentication
permission
model_not_found
rate_limit
context_overflow
timeout
connection
content_filter
invalid_request
provider_internal
cancelled
```

每个错误声明 `retryable`、`retry_after` 和安全消息。原始 Provider 错误只进入受限 Payload。

## 8. 重试与 Fallback

连接失败或限流可以在预算内重试；内容过滤和无效请求不重试。已经向 Agent Loop 交付不可回滚 Tool Call 后不得切换 Provider。Fallback 前重新检查上下文长度、模态、Tool 和 Schema 能力，并记录降级原因。

## 9. 路由

ModelRouter 输入：模式、所需能力、隐私策略、延迟、成本、健康和剩余预算。路由结果包含 `provider_id/model_id/router_policy_version`，并写入 Attempt Trace。

## 10. 配置

Secret 只用引用；Provider 专用选项放在命名空间中并经过 Schema 校验。超时、重试和最大并发有全局默认值和 Provider 覆盖值。

## 11. 契约测试

每个 Adapter 必须通过：文本、结构化输出、单/多 Tool、流事件顺序、取消、超时、限流、Context Overflow、用量统计和原始错误脱敏测试。测试使用固定录制样本，不依赖在线模型稳定回答。
