# 核心领域模型与数据契约

> **文档编号**：DOMAIN-CONTRACTS  
> **适用范围**：Principal、Endpoint、Conversation、Session、Turn、RunAttempt、Task、Delivery、Memory、Envelope 与接口契约  
> **权威边界**：本文是其范围内的规范性来源；总体架构文档只负责概念与边界。  
> **关联文档**：RUNTIME-FLOWS, STORAGE-DATA, ACCESS-DELIVERY

## 阅读说明

**目的**：定义系统共享语言、聚合边界和跨模块契约。

**边界**：不描述具体 Provider 或数据库实现；表结构建议位于 STORAGE-DATA。

**建议读取方式**：Agent 需要生成接口、数据库模型或跨模块变更时应优先加载本文。

## 1. 核心领域模型与聚合边界

### 1.1 建模目的

领域模型不只是数据库表名。它定义：

- 哪些对象具有稳定身份；
- 哪些状态必须在同一事务中保持一致；
- 哪个模块拥有状态变更权；
- 哪些关系是引用而不是嵌套对象；
- 重试、恢复和审计时以什么对象为单位。

### 1.2 核心实体关系

```text
Principal
  └── Endpoint
        └── Conversation
              ├── Session
              ├── Message
              └── Turn
                    └── RunAttempt
                          ├── ToolCall
                          ├── Trace
                          └── Delivery

Task
  └── TaskAttempt
        ├── Checkpoint
        ├── ToolCall
        └── Delivery

Connector
  └── RawItem
        └── Event
              └── ProactiveCandidate
                    └── Decision
                          ├── Task
                          └── Delivery

MemoryItem
  ├── source Message/Event
  ├── supersedes MemoryItem
  └── included in Context Snapshot
```

### 1.3 聚合边界

#### 1.3.1 Conversation Aggregate

包含：

```text
Conversation
Context Partition policy
Partition ordering state
Conversation settings
```

不包含完整 Message 历史；Message 通过 `conversation_id` 引用。

必须保持：

- Context Partition Key 在 reset generation 之间稳定；
- Session 绑定策略合法；
- Conversation 状态与 Endpoint 有效性一致。

#### 1.3.2 Turn Aggregate

包含：

```text
Turn
active RunAttempt reference
priority
status
cancel state
final reply reference
```

RunAttempt 是 Turn 下唯一的执行尝试对象。系统不设置额外的 Run 层；Turn 负责解释逻辑交互是否完成。完整状态和恢复语义见 `EXECUTION-LIFECYCLE`。

#### 1.3.3 Task Aggregate

包含：

```text
Task
current state
schedule
retry policy
lease
checkpoint reference
waiting condition
```

Task 的状态转移必须事务化，Lease 获取和状态检查在同一事务中完成。

#### 1.3.4 Delivery Aggregate

包含：

```text
Delivery
TargetSnapshot
rendered content reference
attempt history
platform receipt
```

Delivery 与 Turn/RunAttempt 解耦，发送失败不回滚已经完成的推理结果。

#### 1.3.5 Memory Aggregate

MemoryItem 是独立事实对象。合并、冲突和替代通过关系字段表达，不把所有记忆塞进一个可变用户画像 JSON。

### 1.4 Principal

表示系统认可的主体。

```text
principal_id
principal_type: owner | external_user | system
status: active | blocked | deleted
created_at
metadata
```

Owner 只有一个并不意味着可以省略 Principal。它是权限、记忆 Scope、审计和身份绑定的根。

### 1.5 Endpoint

表示外部身份或可投递端点：

```text
endpoint_id
channel_type
channel_instance_id
platform_account_id
principal_id
capabilities
status
verified_at
```

一个 Principal 可以绑定多个 Endpoint。Endpoint 变更或解绑不得删除历史 Message 的来源证据。

### 1.6 Conversation

表示平台对话容器：

```text
conversation_id
conversation_endpoint_id
platform_conversation_id
conversation_type: private | group | thread | web
principal_scope
context_partition_policy
status
```

群聊中的 `conversation principal_scope` 与消息发送者 Principal 必须分开，不能把群聊整体当成 Owner。

### 1.7 Session

Session 表示短期 Agent 上下文边界，不等同于平台 Conversation。Session 根据 Channel 和平台 Conversation 解析。

```text
session_id
conversation_id
context_partition_key
reset_generation
status
created_at
```

`context_partition_key` 根据平台 Conversation、可选 Thread 和多用户隔离策略计算；完整规则见 `SESSION-CONTEXT`。

策略：

```text
isolated
  每个 Conversation 独立短期历史

shared_profile
  共享长期 Memory/Preference，不共享短期消息

```

系统不合并不同 Channel 的短期 Session。不同 Session 可以共享经过 Scope 校验的长期 Memory、Preference 和 Goal，不能共享近期 Message、Conversation Summary 或 Context Snapshot。具体规则见 `SESSION-CONTEXT`。

### 1.8 Message

标准化消息：

```text
message_id
conversation_id
session_id
sender_principal_id
sender_endpoint_id
role: user | assistant | tool | system
direction: inbound | outbound | internal
content_parts
reply_to_message_id
platform_message_id
current_revision_no
receive_sequence
trust_label
raw_payload_ref
created_at
```

Message 创建后内容不可原地覆盖；编辑消息使用版本或新 MessageRevision。

### 1.9 Turn 与 RunAttempt

Turn 是一次用户意图的逻辑生命周期，RunAttempt 是一次实际执行尝试。系统没有独立 Run 层。

```text
Turn 1
├── RunAttempt 1  model timeout
└── RunAttempt 2  completed
```

失败、等待后恢复和显式重试都创建新的 RunAttempt，不复活旧 Attempt。字段、状态机、Checkpoint 和恢复规则见 `EXECUTION-LIFECYCLE`。

### 1.10 Task

Task 是可持久恢复的工作单位：

```text
task_id
task_type
payload_ref
status
priority
scheduled_at
retry_policy
lease_owner
lease_expires_at
checkpoint_ref
idempotency_key
origin
```

TaskAttempt 表示一次 Worker 执行权：

```text
task_attempt_id
task_id
attempt_no
status: created | running | succeeded | failed | cancelled | abandoned
lease_owner
lease_version
lease_expires_at
checkpoint_ref
started_at
finished_at
```

Task 上的 Lease 字段是当前执行权快照；具体 Attempt 字段、状态和恢复语义以 `TASK-SCHEDULER` 为准。

### 1.11 Event

Event 是已经发生的不可变事实：

```text
event_id
event_type
aggregate_type
aggregate_id
aggregate_version
payload_ref
occurred_at
ingested_at
content_hash
trust_label
schema_version
```

Event 修正通过新 Event，例如 `MemoryCorrectionApplied`，不修改旧 Event。

### 1.12 Delivery

尚未到发送时机的主动通知使用 `ScheduledDeliveryRequest` 保存内容、建议目标、到期时间和决策原因；它不是 Delivery，也不固定最终 Endpoint。

Delivery 在 ready-to-send Policy 完成后创建，保存固定目标快照、发送意图和结果：

```text
delivery_id
target_snapshot
content_ref
status
idempotency_key
scheduled_at
platform_message_id
last_error
```

Target 使用快照，避免 Delivery 执行期间 Endpoint 配置变化导致发送到意外目标。若 ScheduledDeliveryRequest 等待期间偏好变化，以创建 Delivery 时的 Policy 为准。

### 1.13 MemoryItem

```text
memory_id
kind: fact | preference | episode | goal | constraint
subject
predicate
value
scope
source_type
source_id
confidence
status: candidate | confirmed | rejected | expired
valid_from
valid_to
supersedes_id

# 仅 kind=goal 使用的结构化字段
goal_status: active | paused | completed | cancelled | expired
goal_priority
goal_deadline
goal_progress
```

MemoryItem 必须保留来源，不能只保存模型总结后的无来源文本。

### 1.14 Trace、Span 与 Audit

Trace 表示端到端因果链，Span 表示执行步骤，Audit 表示敏感控制和状态变更记录。

它们不是同一概念：

```text
Trace  调试性能和调用链
Span   Trace 中的局部步骤
Audit  谁在何时以何权限改变了什么
```

## 2. 数据契约与接口架构

### 2.1 契约分类

```text
Cross-process Contract   Gateway、API、Worker、Interaction 之间
Application Contract     Command、Query、Result
Domain Value Object      进程内领域值对象
Plugin API Contract      插件公开接口
Provider Contract        模型、存储、Channel 等实现接口
```

不要把数据库 ORM Entity 直接用作任何公开契约。

### 2.2 契约通用字段

所有跨进程 Envelope 至少包含：

```text
schema_version
message/request/event id
created_at / occurred_at
trace_context
origin
metadata
```

大型内容使用：

```text
inline_small_payload
或
payload_ref + sha256 + content_type + size
```

### 2.3 TraceContext

```text
trace_id
span_id
parent_span_id
correlation_id
causation_id
principal_id
conversation_id
session_id
turn_id
attempt_id
task_id
```

TraceContext 字段允许为空，但一旦设置，不允许普通 Middleware 修改。

### 2.4 ChannelEnvelope

```text
schema_version
message_id
channel_type
channel_instance_id
platform_sender_id
sender_endpoint_ref
conversation_endpoint_ref
platform_conversation_id
thread_id
content_parts
platform_message_id
reply_route
received_at
trust_label
trace_context
metadata
```

`reply_route` 至少包含：

```text
channel_type
channel_instance_id
platform_conversation_id
thread_id
reply_to_platform_message_id
reply_token
reply_token_expires_at
```

### 2.5 ContentPart

统一内容类型：

```text
TextPart
ImagePart
FilePart
AudioPart
VideoPart
LocationPart
QuotePart
ButtonActionPart
StructuredPart
```

每个 Part 包含：

```text
part_id
content_type
inline_data or payload_ref
size
sha256
metadata
trust_label
```

字段边界：`content_type`、`size`、`sha256`、`metadata`、`trust_label` 在跨进程
和持久化契约中均为必填且不可为 `null`。未知或纯内联内容使用
`size=0`、`sha256=""`、`metadata={}`；可选的是 `inline_data` 与 `payload_ref`
二选一的值，不以 SQL `NULL` 表示数值默认。Gateway V0/V1 解码与 Adapter
归一化必须在进入 Core 前完成该默认值转换。

### 2.6 AgentRequest 与 AgentReply

```text
AgentRequest
- turn_id
- principal
- conversation
- context_snapshot
- input_message
- capability_policy
- resource_budget
- trace_context

AgentReply
- turn_id
- content_parts
- reply_mode
- render_hints
- memory_candidates
- suggested_tasks
- status_summary
- trace_context
```

`AgentReply` 不包含平台 SDK 对象。

### 2.7 ToolRequest 与 ToolResult

```text
ToolRequest
- tool_call_id
- tool_name
- tool_version
- arguments
- requested_permissions
- idempotency_key
- timeout
- risk_context
- trace_context

ToolResult
- tool_call_id
- status
- structured_output
- output_ref
- error
- side_effect_receipt
- started_at
- completed_at
```

### 2.8 Command 契约

Command 必须描述意图，不描述数据库操作。

Command 的通用 Envelope、调用方幂等键和 `expected_version` 以 `APPROVAL-COMMANDS / 2. Command Envelope` 为准。

正确：

```text
CancelTurn(turn_id, reason)
ApproveToolCall(approval_id)
RetryDelivery(delivery_id)
LinkEndpoint(endpoint_id, principal_id)
```

错误：

```text
UpdateRunStatusToCancelled
InsertDeliveryRow
SetMemoryStatusField
```

### 2.9 EventEnvelope

```text
event_id
event_type
source
aggregate_type
aggregate_id
aggregate_version
occurred_at
ingested_at
payload_ref
content_hash
trust_label
schema_version
origin
correlation_id
causation_id
trace_context
```

### 2.10 ErrorEnvelope

```text
error_code
category
message_safe
retryable
retry_after
provider_error_ref
field_errors
trace_id
```

公开接口返回安全消息；原始 Provider 错误放在受限 Trace/Payload 中。

### 2.11 Schema 版本策略

- 增加可选字段属于向后兼容；
- 删除或改变字段语义需要新版本；
- Gateway 与 Core 至少支持当前和前一个兼容版本；
- Event Consumer 必须记录已支持版本；
- 插件 Manifest 声明 `api_version` 范围；
- 数据库 Migration 与消息 Schema Version 分开管理。

### 2.12 Envelope 不可变与 Patch

跨模块 Envelope 使用不可变对象。允许修改时返回显式 Patch：

```text
SetField
RemoveField
AppendContentPart
AppendPromptSection
AddMetadata
AddTag
Reject
RequireApproval
```

受保护字段：

```text
trace_id
principal_id
conversation_id
turn_id
attempt_id
origin
reply_route
schema_version
idempotency_key
```

Runtime 负责校验 Patch、应用 Patch、记录 Diff 和来源。

## 3. 关键接口示例

### 3.1 TurnService

```python
class TurnService(Protocol):
    async def accept(self, envelope: ChannelEnvelope) -> TurnAccepted: ...
    async def cancel(self, turn_id: str, reason: str) -> None: ...
    async def resume(self, turn_id: str, command: ResumeCommand) -> None: ...
```

### 3.2 MemoryService

```python
class MemoryService(Protocol):
    async def retrieve(self, query: MemoryQuery) -> MemoryResult: ...
    async def propose(self, candidates: list[MemoryCandidate]) -> list[MemoryItem]: ...
    async def confirm(self, memory_id: str) -> MemoryItem: ...
    async def reject(self, memory_id: str) -> None: ...
```

### 3.3 DeliveryService

```python
class DeliveryService(Protocol):
    async def enqueue(self, request: DeliveryRequest) -> DeliveryRef: ...
    async def cancel(self, delivery_id: str) -> None: ...
    async def retry(self, delivery_id: str) -> None: ...
```

### 3.4 EventPublisher

```python
class EventPublisher(Protocol):
    def publish(self, event: DomainEvent) -> None: ...
```

该调用只将 Event 加入当前 Unit of Work，由 Outbox 在 Commit 后投递。
