# 测试、回放与质量评估

## 1. 测试分层

```text
unit          领域规则、状态机、打分、纯函数
contract      Bridge、Provider、Tool、Plugin、Event Schema
integration   SQLite、Payload、Outbox、Lease、Delivery
recovery      崩溃窗口、unknown、Checkpoint、备份恢复
security      Policy、Sandbox、注入、路径和网络逃逸
replay        历史输入在禁副作用环境重放
evaluation    Agent 行为质量和回归
```

## 2. 确定性设施

测试使用固定时钟、ID、随机种子、临时目录、Stub Provider、Fake Gateway 和 Tool Simulator。不得依赖在线模型文本完全一致。

## 3. 故障注入点

- SQLite Commit 前后；
- Payload rename/metadata 前后；
- Tool 外部成功与 Receipt 写入之间；
- Delivery 发送与结果上报之间；
- Lease 续期、Cursor 推进和 Outbox 发布；
- Approval 响应与过期同时发生。

每个故障点验证恢复后状态和副作用次数。

## 4. Contract Fixture

保存去敏的 LangBot Event、模型响应流、Tool/MCP 消息和旧版本 Event。Adapter 更新必须运行全部 Fixture，并验证字段、错误和顺序。

## 5. 回放

回放读取归档 Message/Event/Context Snapshot，固定配置和模型 Stub。默认禁用真实 Tool、Delivery、Connector acknowledge；输出新的 Trace 和差异报告，不写生产 Inbox。

## 6. Agent 评测集

至少覆盖：普通问答、连续对话、Tool 选择、拒绝越权、长期记忆写入/召回、矛盾事实、主动推送、任务恢复、Context Overflow 和模型降级。

## 7. 指标

```text
task_success
groundedness/source correctness
tool_selection/argument validity
policy_violation
memory_precision/recall
duplicate_side_effect
latency/token/cost
user correction rate
```

LLM Judge 只能辅助，关键权限、幂等和状态断言使用确定性检查。

## 8. 发布门禁

所有单元/契约通过；关键恢复和 Sandbox 用例通过；数据库 Migration 从上一版本成功；核心评测无显著回归；新增已知风险有 ADR 或显式接受记录。

## 9. 验收矩阵

每份功能规范的“验收测试”映射到测试 ID。没有测试或人工演练步骤的状态机、恢复保证和安全约束不得标记完成。

## 10. 数据治理

生产样本进入 Fixture 前脱敏；用户私有内容默认不上传；失败样本保存来源和许可；评测结果记录应用、配置、Prompt、模型和数据集版本。
